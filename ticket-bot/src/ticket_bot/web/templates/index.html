VieShow takeover-first

先選場次，再接手，直接跑到付款頁

先在 VieShow 頁面完成登入 / Cloudflare / 場次選擇，再由 bot 接手抓場次、選票種、選座位，最後停在付款頁。傳統 run / watch 保留在 fallback。

接手模式

操作：開啟 Chrome → 停在 booking.aspx 或 ticket.aspx → 重新抓場次 → 選定場次 → 開始接手。

1
開啟 Chrome
確認 remote debugging port 已啟動，並先登入 / 過驗證。
2
抓場次 / 選場次
從 ticket.aspx 讀出可點場次，選好要搶的那個。
3
開始接手
bot 會帶著你選的場次 id / value 進 takeover 流程。
抓到的場次會直接進下拉選單，開始接手時會一併送到 `/api/start`。
fallback / watch
如果 takeover 不適合目前頁面，可以退回傳統流程。這些欄位只給舊模式用。
